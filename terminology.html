  • Predicate

    Smallest expression part. In algebra it would be a term. operator MUST be one of Predicate operators. see code for current list but includes: $eq, $lt, $lte, $gt, $gte, $like, $anyOf, $oneOf, 
            
          {
            subjectId: 'lastName',
            operator:  '$eq',
            value: 'Flintstone'
          }

  • PredicateBranch

    Conjunction or Disjunction expression. operator MUST be one of PredicateBranch operators. See code for current list but includes: $or, $and 
            
          {
            subjectId: 'lastName',
            operator:  '$eq',
            value: 'Flintstone'
          }

  • Predicate Tree Document

    JSON document in the form of: 
              
            {
              predicateId001:{parentId: null, operator: $and},
              predicateId002:{parentId: 'predicateId001', subjectId: 'lastName', operator: '$eq', value: 'Flintstone'},
              predicateId003:{parentId: 'predicateId001', subjectId: 'firstName', operator: '$eq', value: 'Fred'},
            }
          
          
      
            
    • key/id - no special requirement other than they are logically coherent (parentId is in document)
      • 
            
    • All branches MUST have two or more children
      • 
            
    • One and ONLY one root node {parentId: null,...} 
      • 
            
    • All branches must have parent in document, except root
